Anything Search Privacy Policy

Effective date: July 1, 2026

Data Collection

Anything Search does not collect personal data in the current macOS release.

The app stores a local index on your Mac containing file names, folder names, paths, extensions, file or folder type, size, and timestamps for folders you choose to index. This index remains on your device.

No Tracking

Anything Search does not track you across apps or websites. It does not use third-party advertising SDKs, analytics SDKs, or tracking identifiers.

No Cloud Indexing

Anything Search does not upload your files, file names, folder names, paths, search queries, or local index to a server. Core search works without a network connection.

User-Selected Locations

Anything Search indexes only locations you choose or authorize. You can remove indexed locations from Settings. The App Store build uses macOS sandbox permissions and user-selected folder access.

File Contents

The current release does not index file contents. It indexes file names, paths, and basic metadata only.

Diagnostics And Support

The app may show local diagnostics to help explain search results or permission problems. Diagnostics are generated on your Mac. If you contact support, only the information you choose to send is shared.

Data Deletion

You can remove indexed folders or reset local index data from the app. Removing the app may leave local support files in the standard macOS Application Support location until you delete them manually.

In-App Purchases

If the Mac App Store build offers Anything Pro, purchase processing for the non-consumable product app.anything.pro.unlock is handled by Apple through StoreKit.

The app stores only a local entitlement cache needed to unlock Pro features offline. That cache does not include file names, paths, search text, indexed content, or Apple ID email addresses.

Changes

If future versions add optional online features such as crash reporting, telemetry, license activation, or cloud sync, this policy and the App Store privacy details will be updated before release.
